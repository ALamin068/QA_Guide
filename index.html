🚀 Getting Started - Your QA Journey

👋 Welcome to Quality Assurance!

Don't worry if you feel overwhelmed - every expert was once a beginner. This guide will walk you through each step, explain everything clearly, and help you when you get stuck.

🎯 What is Quality Assurance?

Simple Definition: QA ensures software works correctly before customers use it.


Your Role: You're like a detective who finds problems and a guardian who prevents bad software from reaching users.

📋 Daily QA Activities:

  • 🔍 Testing: Click buttons, fill forms, try to break things
  • 📝 Documenting: Write down what you tested and what you found
  • 🐛 Bug Reporting: Tell developers what's broken and how to fix it
  • 🤖 Automation: Write code to test things automatically
  • 👥 Collaboration: Work with developers, product managers, and other testers

📋 Manual Testing Practices

🎯 Manual Testing Made Simple

What is Manual Testing? It's testing software by hand - clicking buttons, filling forms, and checking if everything works as expected. Think of it like being a user, but with a detective's mindset!

🤔 Why Manual Testing Matters:

  • ✅ Finds issues automation might miss
  • ✅ Tests user experience and usability
  • ✅ Doesn't require coding skills to start
  • ✅ Helps you understand the application deeply

📝 Writing Your First Test Case

Step 1: Understand the Requirements

Before writing any test: Read the feature requirements. Ask yourself: "What is this supposed to do?"

Example: "Users should be able to login with their email and password"

Step 2: Think Like a User

Happy Path: Everything goes right (user enters correct details)

Sad Path: Things go wrong (user enters wrong password)

Edge Cases: Unusual situations (user enters emoji in password)

Step 3: Write the Test Case

📋 Test Case Template (Copy This!):
Test Case ID: TC_[FEATURE]_[NUMBER]
Test Title: [Action] should [Expected Result]
Priority: High/Medium/Low
Pre-conditions: What needs to be ready before testing
Test Data: What data you'll use
Test Steps:
1. [Detailed action]
2. [Detailed action]
3. [Detailed action]
Expected Result: What should happen
Actual Result: [Fill during execution]
Status: [Pass/Fail/Blocked]
Comments: [Any notes or observations]

🔍 Testing Techniques for Beginners

1. Boundary Value Analysis (Simple Version)

What it means: Test the edges of what's allowed

Example: If age field accepts 18-65, test: 17 (too low), 18 (minimum), 65 (maximum), 66 (too high)

🎯 Try This:

Field: Password must be 8-20 characters

Test Values:

  • 7 characters: "pass123" (should fail)
  • 8 characters: "pass1234" (should pass)
  • 20 characters: "pass123456789012345" (should pass)
  • 21 characters: "pass1234567890123456" (should fail)

2. Equivalence Partitioning (Simple Version)

What it means: Group similar inputs together and test one from each group

Example: For age groups, test one teenager (16), one adult (25), one senior (70)

3. Error Guessing (Detective Work)

Always test these:

  • 🔍 Empty fields: What happens if you leave it blank?
  • 🔍 Wrong data type: Put letters where numbers are expected
  • 🔍 Special characters: Try @#$%^&* in text fields
  • 🔍 Very long text: Copy-paste a paragraph into a name field
  • 🔍 Spaces: Start or end with spaces
  • 🔍 Case sensitivity: Try UPPERCASE, lowercase, MiXeD

🚀 Before You Start Testing

🎯 During Testing

Step 1: Execute Test Cases
  • Follow test steps exactly as written
  • Take screenshots of important steps
  • Note actual results as you go
  • Don't skip steps (even if they seem obvious)
Step 2: Compare Results
  • Does actual result match expected result?
  • If yes → Mark as PASS
  • If no → Mark as FAIL and document the issue
  • If you can't complete → Mark as BLOCKED

  • Try related functionality
  • Look for visual issues
  • Check loading times
  • Test on different screen sizes

🐛 Bug Reporting for Beginners

❓ What Makes a Good Bug Report?

  • ✅ Clear title: Describes the problem in one sentence
  • ✅ Steps to reproduce: Anyone can follow and see the same issue
  • ✅ Expected vs Actual: What should happen vs what really happened
  • ✅ Environment details: Browser, OS, device used
  • ✅ Screenshots/videos: Visual proof of the issue

🚨 How to Choose Bug Severity

🔴 Critical: App crashes, data loss, security breach

Example: "App crashes when user clicks submit button"

🟠 High: Major feature completely broken

Example: "Users cannot login at all"

🟡 Medium: Feature works but has issues

Example: "Search results are sometimes incorrect"

🟢 Low: Cosmetic issues, typos

Example: "Button text has spelling error"

❌ Common Beginner Mistakes & How to Avoid Them

❌ Mistake 1: Vague Bug Reports

Bad: "Login doesn't work"

Good: "Error message 'Invalid credentials' appears when entering correct username and password on login page"

💡 Fix: Always include specific details about what you did and what happened

❌ Mistake 2: Not Following Test Steps Exactly

Bad: Skipping steps because "they seem obvious"

Good: Following every step exactly as written

💡 Fix: Test cases are written for a reason. Each step might reveal different issues

❌ Mistake 3: Only Testing Happy Path

Bad: Only testing when everything goes right

Good: Testing both success and failure scenarios

💡 Fix: Always ask "What if something goes wrong?" and test those scenarios

❌ Mistake 4: Not Taking Screenshots

Bad: Describing visual issues with words only

Good: Including screenshots of every bug found

💡 Fix: A picture is worth a thousand words. Always include visual proof

❓ Manual Testing FAQ

❓ How do I know if I found a real bug or if it's expected behavior?

Answer: Check the requirements first. If behavior doesn't match requirements, it's likely a bug. When in doubt, ask your team lead or product owner.

Tip: It's better to report something that might be a bug than to miss a real issue.

❓ How detailed should my test steps be?

Answer: Detailed enough that someone who has never seen the application could follow them and get the same result.

Example: Instead of "login", write "Enter 'testuser@example.com' in the email field, enter 'password123' in the password field, click the blue 'Login' button"

❓ What if I can't reproduce a bug I found earlier?

Answer: This happens! Try these steps:

  1. Try the exact same steps again
  2. Check if anything changed (different browser, data, etc.)
  3. Try on a different device/browser
  4. Document that it's intermittent and when you saw it
❓ How do I prioritize which bugs to report first?

Answer: Report in this order:

  1. Critical: Crashes, data loss, security issues
  2. High: Core features completely broken
  3. Medium: Features work but have problems
  4. Low: Cosmetic issues, typos
❓ Should I test on multiple browsers?

Answer: Yes! Different browsers can show different issues. Start with Chrome, then test Firefox and Safari. Check requirements for which browsers your app should support.

❓ What if I'm not sure how something should work?

Answer: Don't guess! Ask your team:

  • Check requirements documentation
  • Ask the product owner or business analyst
  • Look at similar features in the app
  • Check how competitors handle it

  • Test minimum values (0, 1)
  • Test maximum values (99, 100)
  • Test just outside boundaries (-1, 101)

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.chrome.ChromeOptions;
import io.github.bonigarcia.wdm.WebDriverManager;

public class DriverFactory {
    public static WebDriver createDriver(String browser) {
        WebDriver driver;

        switch (browser.toLowerCase()) {
            case "chrome":
                WebDriverManager.chromedriver().setup();
                ChromeOptions options = new ChromeOptions();
                options.addArguments("--headless"); // For CI/CD
                options.addArguments("--no-sandbox");
                options.addArguments("--disable-dev-shm-usage");
                driver = new ChromeDriver(options);
                break;
            case "firefox":
                WebDriverManager.firefoxdriver().setup();
                driver = new FirefoxDriver();
                break;
            default:
                throw new IllegalArgumentException("Browser not supported: " + browser);
        }

        driver.manage().window().maximize();
        driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
        return driver;
    }
}

Page Object Model Example

// BasePage.java
public class BasePage {
    protected WebDriver driver;
    protected WebDriverWait wait;

    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        PageFactory.initElements(driver, this);
    }

    protected void waitAndClick(By locator) {
        WebElement element = wait.until(ExpectedConditions.elementToBeClickable(locator));
        element.click();
    }

    protected void waitAndSendKeys(By locator, String text) {
        WebElement element = wait.until(ExpectedConditions.presenceOfElementLocated(locator));
        element.clear();
        element.sendKeys(text);
    }
}

// LoginPage.java
public class LoginPage extends BasePage {
    private static final By USERNAME_FIELD = By.id("username");
    private static final By PASSWORD_FIELD = By.id("password");
    private static final By LOGIN_BUTTON = By.cssSelector("button[type='submit']");
    private static final By ERROR_MESSAGE = By.cssSelector(".alert-danger");

    public LoginPage(WebDriver driver) {
        super(driver);
    }

    public LoginPage enterUsername(String username) {
        waitAndSendKeys(USERNAME_FIELD, username);
        return this;
    }

    public LoginPage enterPassword(String password) {
        waitAndSendKeys(PASSWORD_FIELD, password);
        return this;
    }

    public DashboardPage clickLogin() {
        waitAndClick(LOGIN_BUTTON);
        return new DashboardPage(driver);
    }

    public String getErrorMessage() {
        return wait.until(ExpectedConditions.presenceOfElementLocated(ERROR_MESSAGE)).getText();
    }

    // Fluent interface for chaining
    public DashboardPage login(String username, String password) {
        return enterUsername(username)
                .enterPassword(password)
                .clickLogin();
    }
}

Mobile Test Patterns

// Base Mobile Test Class
public class BaseMobileTest {
    protected AppiumDriver driver;

    @BeforeMethod
    public void setUp() {
        driver = MobileDriverFactory.createAndroidDriver();
        driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
    }

    @AfterMethod
    public void tearDown() {
        if (driver != null) {
            driver.quit();
        }
    }

    protected void waitForElement(By locator, int timeout) {
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(timeout));
        wait.until(ExpectedConditions.presenceOfElementLocated(locator));
    }

    protected void hideKeyboard() {
        try {
            ((AndroidDriver) driver).hideKeyboard();
        } catch (Exception e) {
            // Keyboard not present
        }
    }
}

// Page Object for Mobile
public class LoginScreen extends BaseMobileTest {
    private static final By USERNAME_FIELD = AppiumBy.id("com.example:id/username");
    private static final By PASSWORD_FIELD = AppiumBy.id("com.example:id/password");
    private static final By LOGIN_BUTTON = AppiumBy.accessibilityId("Login Button");

    public LoginScreen enterUsername(String username) {
        waitForElement(USERNAME_FIELD, 10);
        driver.findElement(USERNAME_FIELD).sendKeys(username);
        return this;
    }

    public LoginScreen enterPassword(String password) {
        driver.findElement(PASSWORD_FIELD).sendKeys(password);
        hideKeyboard();
        return this;
    }

    public HomeScreen clickLogin() {
        driver.findElement(LOGIN_BUTTON).click();
        return new HomeScreen();
    }
}

Database Connection in Java

// Database connection utility
import java.sql.*;

public class DatabaseUtil {
    private static final String URL = "jdbc:mysql://localhost:3306/testdb";
    private static final String USERNAME = "testuser";
    private static final String PASSWORD = "testpass";

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USERNAME, PASSWORD);
    }

    // Execute SELECT query and return results
    public static ResultSet executeQuery(String query) throws SQLException {
        Connection conn = getConnection();
        Statement stmt = conn.createStatement();
        return stmt.executeQuery(query);
    }

    // Execute INSERT/UPDATE/DELETE and return affected rows
    public static int executeUpdate(String query) throws SQLException {
        try (Connection conn = getConnection();
             Statement stmt = conn.createStatement()) {
            return stmt.executeUpdate(query);
        }
    }

    // Prepared statement for safe parameter binding
    public static ResultSet executeQuery(String query, Object... params) throws SQLException {
        Connection conn = getConnection();
        PreparedStatement pstmt = conn.prepareStatement(query);

        for (int i = 0; i < params.length; i++) {
            pstmt.setObject(i + 1, params[i]);
        }

        return pstmt.executeQuery();
    }
}

// Example test class using database validation
public class DatabaseTest {

    @Test
    public void testUserCreation() throws SQLException {
        // Create test user
        String insertUser = "INSERT INTO users (username, email) VALUES (?, ?)";
        DatabaseUtil.executeUpdate(insertUser, "testuser", "test@example.com");

        // Verify user was created
        String selectUser = "SELECT * FROM users WHERE username = ?";
        ResultSet rs = DatabaseUtil.executeQuery(selectUser, "testuser");

        assertTrue(rs.next());
        assertEquals("testuser", rs.getString("username"));
        assertEquals("test@example.com", rs.getString("email"));

        // Cleanup
        String deleteUser = "DELETE FROM users WHERE username = ?";
        DatabaseUtil.executeUpdate(deleteUser, "testuser");
    }
